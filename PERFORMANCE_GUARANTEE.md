| `robots.txt` | 549B | ❌ 使用者不會下載 |

**關鍵**：
- `sitemap.xml` 和 `robots.txt` 僅供搜尋引擎爬蟲使用
- 使用者瀏覽網站時完全不會載入這兩個檔案

### 2. **Head 區段優化**

```

  優化後: 1 個 script 標籤，927 bytes
  減少:   60% 體積
```

**技術細節**：
- `type="application/ld+json"` 不是可執行的 JavaScript
- 瀏覽器不會執行這段代碼
- 僅作為資料供搜尋引擎讀取
- 解析時間 < 2ms（幾乎為零）

### 3. **Meta 標籤特性**

```html
<meta name="keywords" content="...">
<meta name="description" content="...">
<meta property="og:..." content="...">
```

**特點**：
- 純文字，無需處理
- 瀏覽器解析時間 < 1ms
- 不影響 DOM 構建
- 不阻塞渲染

## 🔬 技術原理說明

### 為什麼 JSON-LD 不影響效能？

```html
<script type="application/ld+json">
{"@context":"https://schema.org",...}
</script>
```

1. **不是 JavaScript 代碼**
   - `type="application/ld+json"` 告訴瀏覽器這是資料，不是代碼
   - 瀏覽器不會執行它

2. **不阻塞渲染**
   - 與 `<script>` 或 `<script defer>` 不同
   - 瀏覽器會跳過它，繼續渲染頁面

3. **僅供搜尋引擎**
   - Google 爬蟲會讀取這段資料
   - 使用者瀏覽器會忽略它

### 為什麼 sitemap.xml 不影響效能？

```
使用者訪問網站:
  1. 瀏覽器請求: index.html
  2. 瀏覽器載入: styles.css, script.js, images/...
  3. ❌ 不會請求 sitemap.xml

搜尋引擎爬取:
  1. Google 爬蟲訪問: robots.txt
  2. 讀取 Sitemap 位置
  3. 單獨爬取: sitemap.xml
  4. ✅ 與使用者完全分離
```
